[fleXive] announcement submission example application

Welcome to the [fleXive] announcement submission example application

What this example application is doing
Announcements like software release messages are submitted on news websites etc.
Every time an announcement is submitted, it is logged in the announcement submission application.
When it got published, the publishing information (date, URL) is logged in addition.
This application supports user authentication and content security.

There have been two user groups set up by the runonce script asubmission001.groovy.

Each user group has a user assigned.

For the first group it's

  • Username: user_01
  • Password: password

For the second group it's

  • Username: user_02
  • Password: password

Users of the first group may read, create, edit and delete content instances.
Users of the second group may only read content instances.
